# PEACOCK DRIVERS WEB APP — COMPLETE REPLIT BUILD INSTRUCTIONS

You are building the UI for a Sri Lanka travel booking web app for Peacock Drivers. This is a high-fidelity prototype with mocked data — no backend integrations yet. The code will later be exported to Claude Code for wiring up Stripe, auth, database, and maps.

## CRITICAL: Export-friendly conventions

This project WILL be exported to another environment. Follow these rules strictly:

1. **Framework:** Next.js 14 with App Router (NOT Pages Router)
2. **Styling:** Tailwind CSS only (no CSS modules, no styled-components)
3. **Icons:** Lucide React only (`lucide-react`)
4. **Components:** PascalCase, in `/components` folder
5. **Pages:** In `/app` folder following the route structure below
6. **Mock data:** JSON files in `/data` folder, imported directly
7. **No Replit-specific APIs or dependencies**
8. **No localStorage for state** — use React state/context
9. **No authentication logic** — just mock the logged-in state with context

### Fonts

**Display: Instrument Serif**
- Import: `@import url('https://fonts.googleapis.com/css2?family=Instrument+Serif:ital@0;1&display=swap');`
- Usage: ALL display headings, section titles, hero text, tour names, card titles
- Weights: 400 (regular), 400 italic
- KEY PATTERN: One word per heading set in italic for editorial warmth. E.g., "See our amazing *journeys*", "Witness Sri Lanka's art of *hospitality*". This is a signature brand touch — carry it into every section heading.

**Body: DM Sans**
- Import: `@import url('https://fonts.googleapis.com/css2?family=DM+Sans:ital,opsz,wght@0,9..40,400;0,9..40,500;0,9..40,600;0,9..40,700;1,9..40,400&display=swap');`

### UI Patterns (from the marketing site — replicate these exactly)

**1. Section structure:**
Every major section follows: overline label (uppercase sans, text-xs, tracking-widest, text-amber-300 or text-warm-400) → serif heading (font-display, with one italic word) → body text → content. Sections alternate between bg-white and bg-cream.

**2. Product tabs:**
Pill tabs in a neutral track. Active tab = bg-forest-500 text-white rounded-full. Inactive = transparent text-warm-500. Track = bg-warm-50 rounded-full p-1.

**3. Tour/journey cards:**
Full-bleed image with gradient overlay (transparent at top → forest-500/70 at bottom). Location name in font-display text-white, description in font-body text-white/80. Below image: duration tag, category chips, starting price. Card = rounded-2xl shadow-card.

**4. Vehicle selector cards:**
Centered vehicle image, name below (font-body font-semibold), capacity text (font-body text-sm text-warm-400), daily price (font-body font-semibold text-forest-500). Selected state: ring-2 ring-forest-500 bg-forest-50.

**5. Buttons:**
- Primary: bg-forest-500 text-white rounded-full hover:bg-forest-400. Sizes: sm (py-2 px-5), md (py-3 px-7), lg (py-4 px-9).
- Secondary: bg-transparent text-forest-500 border-[1.5px] border-forest-500 rounded-full hover:bg-warm-50.
- Ghost: bg-transparent text-forest-500 rounded-full hover:bg-warm-50.
- Amber: bg-amber-300 text-white rounded-full hover:bg-amber-200. For payment CTAs.

**6. Status badges:**
Pill-shaped (rounded-full), small (text-xs px-3 py-1 font-medium):
- Confirmed: bg-[#E8F5E9] text-[#2D7A4F]
- Pending / Quote Requested: bg-amber-50 text-amber-400
- Cancelled: bg-[#FDECEA] text-[#C4382A]
- Completed: bg-warm-50 text-warm-500
- In Progress: bg-[#E6F1FB] text-[#2A6B8C]

**7. Form inputs:**
rounded-lg (8px), border-[1.5px] border-warm-100, py-3 px-4, font-body text-[15px]. Focus: border-forest-500 ring-0 outline-none. Label: font-body text-[13px] font-medium text-warm-600 mb-1.5.

**8. Dark CTA section:**
bg-forest-500, font-display text-white heading (with italic word), body text text-white/70, primary button with bg-white text-forest-500.

**9. Trust bar:**
Horizontal row of press logos in grayscale/muted treatment. Subtle, not loud.

**10. FAQ accordion:**
font-display for question text, expand/collapse chevron icon, answer in font-body.
